# Lazarus — Impeccable Design Context

## Design Context

### Users
Hackathon judges and pharma executives watching a live demo. They see this once, for 3–5 minutes, on a large screen. The interface needs to look immediately impressive and feel like a real operations center — not a side project. The job-to-be-done is "make the AI pipeline feel alive and credible."

### Brand Personality
**Surgical. Relentless. Alive.**
A military-grade bioinformatics operations center. The feeling of watching a real autonomous system work. Not a dashboard — a command terminal for resurrecting dead drugs.

### Aesthetic Direction
- **Primary reference:** The Three.js static dashboard (`/tmp/lazarus-threejs.html`) — dark terminal HUD with rotating particle globe, scanline overlay, agent log feed, color-coded agent roles, all-caps monospace labels
- **Anti-references:** Modern SaaS (rounded white cards, blue gradients, Inter font, light mode, Tailwind default aesthetics). The current React dashboard leans too far this direction and needs to be pulled hard toward the Three.js reference.
- **Theme:** Dark only. Background `#030712` (near-black). No light mode.
- **The globe:** The Three.js rotating particle sphere must be embedded in the React app — as a persistent background or hero element, not a separate file.

### Design Tokens (match the Three.js dashboard exactly)
```
--bg:       #030712
--panel:    rgba(3, 9, 22, 0.82)
--border:   rgba(16, 185, 129, 0.18)   ← green-tinted, not blue
--green:    #10b981                     ← primary accent
--purple:   #8b5cf6                     ← advocate agent
--red:      #ef4444                     ← skeptic agent
--amber:    #f59e0b                     ← judge agent
--cyan:     #06b6d4                     ← system / openclaw
--text:     #94a3b8
--bright:   #e2e8f0
```

### Typography
- **Font:** `JetBrains Mono` exclusively — monospace everywhere, including body text, labels, values, and nav
- **Labels:** ALL CAPS, letter-spacing 0.18–0.35em, font-size 8–9px
- **Values:** 14–18px, font-weight 700, color `--green` for highlighted metrics
- **No Inter.** Remove it.

### Agent Color System (must be consistent across all components)
| Agent | Color | Usage |
|---|---|---|
| Advocate | `#c4b5fd` / purple | Border-left on log entries, badges |
| Skeptic | `#fca5a5` / red | Border-left on log entries, badges |
| Judge | `#fcd34d` / amber | Border-left on log entries, badges |
| System / OpenClaw | `#10b981` / green | Status, system messages |
| Evidence / Curator | `#67e8f9` / cyan | Supporting evidence entries |

### Layout
- **Left panel:** Three.js globe (persistent, full-height or hero), agent log feed below it
- **Right panel:** Metrics, pipeline steps, hypothesis output, blueprint status
- Current sidebar + tab nav structure can stay but must be restyled to match the terminal aesthetic: tighter, darker, green-accent only, monospace labels
- Scanline overlay: keep it (`repeating-linear-gradient` with 0.04 opacity)
- Scan beam animation: keep it (horizontal green line sweeping top to bottom)

### Design Principles
1. **Everything looks live.** Blinking dots, pulsing glows, animated scan beams. The interface should feel like it's actively running even when idle.
2. **Monospace is non-negotiable.** Every character of text should read like it came out of a terminal. No rounded humanist fonts.
3. **Green means signal, everything else is noise.** Only meaningful data gets green. Muted labels stay slate/dim. Agent-specific colors are the only exception.
4. **Density over whitespace.** Pack information tightly. Generous padding feels like a toy. This is an ops center.
5. **The globe is the hero.** The Three.js particle sphere is the centerpiece — it signals "AI is running." It should be visible at all times, not buried.
