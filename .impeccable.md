# Lazarus — Impeccable Design Context

## Design Context

### Users
Two audiences share this interface:
- **Scientists & researchers** running the pipeline — they need precise system state, live feedback, and trust in the data they're acting on.
- **Executives & investors** reviewing results — they need readable outputs, clear verdicts, and a sense that the tool is credible.

Both groups need to feel like they understand what the system is doing at any moment. Neither should feel lost.

### Brand Personality
**Precise. Authoritative. Alive.**

A serious scientific instrument, not a consumer app or generic SaaS tool. It has opinions. It shows its work. It does not decorate for decoration's sake.

### Emotional Goal
**Confident and in control.** Users should always know where the system is, what it found, and what to do next. The UI earns trust through clarity — not through reassuring copy or visual noise.

### Aesthetic Direction
- **Theme:** Light mode only. Warm beige (`#f7f5f0`) background, forest green (`#1f3a2e`) accent. The current palette is correct — don't introduce new colors without clear purpose.
- **Typography:** Bricolage Grotesque for display headings, Barlow for body text, JetBrains Mono for all technical labels, logs, data, and status. Minimum readable size: 11px. Nothing smaller in production UI.
- **Visual language:** Flat, minimal, precise. 1px borders, 2px radius, no drop shadows except floating panels. Grain texture and subtle grid pattern add depth without ornamentation.
- **Motion:** Purposeful only — state transitions (idle → running → done → failed), drawer entrances, live data updates. No animation that exists purely to feel modern.
- **Density:** Information-dense but not cluttered. Panels breathe with 8px-base spacing. Grid layouts over freeform stacking.
- **Anti-references:** No AI slop — no generic gradients, no glassy frosted cards, no purple/teal AI color schemes, no oversized rounded corners, no chatbot-style UI patterns, no excessive whitespace hiding a lack of content.

### Layout Behavior

- **Sidebar is collapsible.** The left sidebar (globe + agent log feed) is valuable but visually dominant. It must be collapsible via a toggle so users can reclaim space when focused on analysis. Collapsed state: just a narrow strip with a reopen control. State should persist via localStorage.
- **Progressive disclosure, not everything at once.** Tabs should show the most important summary first; details expand on demand. Metrics, confidence gauges, and key verdicts lead — raw logs, full timelines, and detailed breakdowns are secondary and should not compete visually with primary content.
- **Each tab has one primary job.** The Dashboard monitors the live run. Portfolio ranks assets. Compare evaluates hypotheses side-by-side. Strategy maps next steps. Don't let secondary elements (e.g., logs, metadata) dilute the tab's primary purpose.

### Design Principles

1. **Show the system's state, always.** Every panel answers: what is happening right now? Idle, running, done, failed — never ambiguous. Use color, symbols, and status labels consistently.

2. **Earn trust through precision.** Labels are uppercase monospace. Numbers have the right decimal places. Timestamps are real. Output summaries are parsed into readable text — never raw JSON in the UI.

3. **Two audiences, one hierarchy.** Executives read the verdict at the top. Scientists drill into agent traces below. Structure content so both paths are natural — no audience should have to fight the layout.

4. **Restraint over decoration.** If a visual element doesn't carry information or establish hierarchy, remove it. The aesthetic comes from typography, spacing, and color — not from extra UI chrome.

5. **Never leave the page during a live run.** Navigating away during active reasoning kills the SSE stream. All detail views (agent trace, live findings, output) must be accessible as in-page panels or tabs, not new routes.

6. **Reduce to what matters now.** The biggest enemy of this interface is clutter — too many panels competing at once. The sidebar and main content fighting for attention, multiple dense visualizations side-by-side, and collapsed but still-visible secondary info all add cognitive load. Default to less: hide what isn't needed in the current moment, surface it on demand.
